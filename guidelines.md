Assignment Task
Task 1: Process Management and Threading (25 marks)
1.1 Requirements
Develop a multi-threaded application in C/C++ or Python that demonstrates process
creation, thread management, and synchronization. Your application should:
1. Create multiple threads to perform concurrent tasks (minimum 3 threads)
2. Implement proper synchronization mechanisms (mutexes, semaphores, or
monitors)
3. Demonstrate process scheduling by implementing a simple round-robin
scheduler simulation
4. Handle race conditions and demonstrate deadlock prevention
1.2 Deliverables
1. Source code with comprehensive comments
2. Documentation explaining your design decisions (500-750 words)
3. Screenshots or output logs demonstrating successful execution
Task 2: Memory Management Simulation (25 marks)
2.1 Requirements
Create a memory management simulator that implements virtual memory concepts.
1. Implement a paging system with configurable page size
2. Simulate page replacement algorithms (at least two: FIFO and LRU)
3. Track page faults and calculate hit/miss ratios
4. Provide visualization or detailed logging of memory allocation
2.2 Deliverables
1. Source code with detailed comments
2. Test cases demonstrating different page replacement algorithms
3. Analysis report comparing algorithm performance (500-750 words)
Task 3: File System Operations and Security (25 marks)
3.1 Requirements
Develop a secure file management system with the following features:
1. File creation, reading, writing, and deletion operations
2. User authentication mechanism
3. File permission system (read, write, execute for owner/group/others)
4. Encryption/decryption capability for sensitive files
5. Audit logging to track file access and modifications
3.2 Deliverables
1. Source code with security considerations documented
2. User guide explaining how to use the system
3. Security analysis discussing potential vulnerabilities (500-750 words)
Task 4: Network Programming and IPC (25 marks)
4.1 Requirements
Implement a client-server application demonstrating inter-process communication:
1. Create both server and client applications using sockets
2. Implement a simple protocol for data exchange
3. Handle multiple concurrent client connections
4. Implement basic security measures (authentication, data validation)
5. Demonstrate proper error handling and connection management
4.2 Deliverables
1. Source code for both client and server
2. Protocol documentation
3. Testing documentation showing successful communication (500 words)
